BCI-HRP // 人类回归协议

魏珏然(Wei Jueran)· Mellow — 宾夕法尼亚大学教育研究生院 学习科学与技术 M.S.Ed.

一个以协议为先、面向实验室的研究提案,检验声音探针、自我预测、现象学报告与可编辑的 AI 命名,能否支持个体内的自我校准(self-calibration)与情绪-行动校准(affective-action calibration)——自主神经与神经信号仅作可选代理,而非隐藏真相。

当前范围:Phase −1 内部仅暂停设计观察 + Phase 0 Sound-QDR 软件协议。定制 BCI 硬件属未来方向,试点并不需要。

不诊断 不治疗 不诱导状态 非消费级设备 协议为先 独立提案
读 3 分钟简报

一条可检验的主张

在一切之前,先给出一个可被证伪的问题。

"声音探针 + 自我预测 + 现象学报告 + 可编辑命名"这一协议,能否改善个体内的自我校准与情绪-行动校准——以及自主神经/神经代理是否在自我报告之外提供额外的解释力?

不是另一个脑解码器

这不是另一个脑解码器。 BCI-HRP 是一个面向主观经验的校准协议——最终解释权在参与者,而不是信号。

研究立场。首期研究是可行性与构念校准,不是临床验证。"自我校准"与"人类回归"是候选构念——合作实验室在预注册前可以自由收窄、重命名、否决或修订它们。

命名边界

BCI-HRP 在此仅作项目名与长期研究方向。眼下的试点不是定制 BCI 的搭建。第一个实证对象是协议本身:预测 → 探针 → 报告 → 可编辑命名 → 自主性事件。

不是设备推销

首期研究不需要任何定制硬件。

不是读心

代理信号不揭示意义,也不揭示"真实"情绪。

不是固定教条

实验室有权重新设计该构念。

不是治疗

不作任何治疗或危机检测的声称。

研究问题与理由

第一项实验真正要检验什么。

第一项研究不是为了证明临床获益或产品有效性。它要问的是:完整协议是否在普通暂停、自我报告和任务学习之外,提供额外的校准价值。

RQ1 · 可行性

参与者能否完成协议?

参与者能否理解并完成探针前预测、声音探针聆听、探针后现象学报告、可编辑 AI 命名与自主性控件?

RQ2 · 额外价值

完整协议是否提供额外校准价值?

完整 BCI-HRP 是否在 pause-only 与 self-report-only 基线之外,提供可观察的校准价值?

RQ3 · 构念效度

预测变化真的是自我校准吗?

如果预测误差下降,该下降是否同时被感质细粒度、命名稳定性、行动倾向清晰度、对象/契合性报告与自主性事件支持?

RQ4 · 代理价值

可选信号是否增加解释力?

HRV、EDA、EEG、fNIRS 或动作代理是否比第一人称预测和报告本身提供更多解释力?

RQ5 · 安全

自主性控件是否按预期工作?

参与者是否把拒绝、重命名、编辑、删除、停止、回到地面、跳过理解为有意义的行动,而不是错误?

研究理由

情感 BCI、神经反馈与音乐-情绪系统常在估计、分类或调节情感状态。BCI-HRP 走互补路线:它测试一个结构化协议能否帮助参与者预测、描述、修订并拥有自己身体-情绪-行动状态的意义。声音是可重复探针,AI 标签是可编辑假设,自主性事件是一手数据。

分析顺序

试点预先设定 primary → secondary 层级,避免事后挑结果。RQ1(可行性)与 RQ5(安全)是研究闸门;RQ2——相对 pause-only 与 self-report-only 基线的额外价值——是主要推断问题;RQ3(构念效度)与 RQ4(代理价值)只有在 RQ2 模式成立时再读取。混合结果会作为混合结果报告,不事后改写为成功。

定位

不是"音乐 + BCI + 情绪调节",而是以参与者为中心的校准层。

本项目与情感 BCI、脑机音乐接口、EEG/fNIRS 音乐-情绪研究、神经反馈相邻。但它的差异化对象不是新的传感器或新的情绪分类器,而是一个保护解释权、同时让校准可检验的协议。

传统 BCI

信号解码 / 控制

常强调从神经信号中解码外部控制、意图推断或状态估计。

情感计算

情绪分类

常从生理、行为或多模态特征中估计情绪类别、效价或唤醒度。

神经反馈 / BCMI

反馈或调节

常通过声音或音乐反馈训练平静、专注、一致性或自适应情绪调节。

BCI-HRP

参与者自有的校准

把声音用作探针,让参与者先预测、先报告,把 AI 标签视作可编辑假设,并把拒绝/重命名/编辑/删除记录为自主性数据。

操作性差异

BCI-HRP 不问系统能否推断用户的"真实情绪"。它问的是:一个结构化协议能否改善参与者对自身身体-情绪-行动状态的预测、描述、修订与意义归属能力。

对合作实验室

这被设计为一个低风险的 Phase 0 试点:有可复用协议材料、明确失败条件,并围绕情感系统中的解释自主性提出一个可发表研究问题。完整的合作实验室收益说明放在"实验室契合"之后。

为什么重要

多数系统要么过度解读使用者,要么对体验描述不足。

情绪工具常把人塞进"平静""紧张"这类粗标签;消费级神经科技常用薄弱的生理代理过度声称"隐藏洞察";音乐工具常只优化情绪歌单,而不建模一个人的预测、解释控制权或安全退出。本协议瞄准被忽略的中间地带:把声音当作受控探针,让使用者先预测,生理始终只是代理,AI 的解释始终可编辑、可拒绝。

缺口 01

过于依赖粗糙的情绪标签,对感质的细粒度关注太少。

缺口 02

给传感器太多权威,给使用者自有的解释太少。

缺口 03

把声音当情绪调控的太多,把声音当带可证伪结果的探针的太少。

缺口 04

伦理口号太多,交互层面的安全设计太少。

情绪-行动立场

BCI-HRP 不把情绪当作隐藏的内部状态、通用的神经指纹,或一个"越正面越好"的分数。它把"感受到的状态"看作一种具身的、面向行动的、与世界的关系。声音探针被用作可重复的情境,让参与者去觉察身体感受、追踪行动倾向、识别情绪可供性(affordance),并修订自己状态的意义。AI 层是一个可编辑的情绪脚手架,不是诊断权威。

科学分层模型

架构可以丰富,但必须在科学上可读。

一次"声音-反应"会话经过七层,从物理刺激一路到使用者自有的解释。

01

声音事件

真正的刺激:一个固定音频文件,具备有记录的声学属性与事件标记。

02

感官接触

声音如何进入——听觉、身体共振、呼吸耦合、注意捕获。

03

自主神经/身体反应

HRV、EDA、动作、体温与身体自评,作为个体内的反应痕迹。

04

情绪-行动建构

身体变化如何成为压力、释然、激活、安全或淹没感——以及它如何打开或关闭可能的行动。

05

认知预测

使用者在声音之前的预期,以及预测误差是否随会话下降。

06

对象关系与契合性

这份感受是"关于"什么、让什么成为可能、是否契合情境、对象与价值关系。

07

解释自主性

接受、编辑、拒绝、重命名、删除、停止、回到地面、跳过——作为使用者自有的解释控制权。

科学锚点

内感受(Interoception)

身体状态的感知与解释。

稳态调节(Allostasis)

关注调节与恢复,而非静态的内稳态。

建构情绪论

情绪并非从信号中被直接读出。

预测加工

探针前的预测误差是一个真实的可操作目标。

个人信息学

采集 → 反思 → 行动,但不把数据当作绝对真理。

音乐干预的严谨性

刺激应有记录、可复现、声称要窄。

协议快照

一位 PI 应能在三分钟内判断是否契合。

核心单元

会话,而非设备

研究单元是一次"声音-反应"会话:预测、声音探针、报告、可编辑命名与自主性事件。

主检验

预测误差

个体内的探针前预测误差是否随会话变化,并能迁移到记忆之外的片段。

安全原则

先自主,后解释

AI 摘要始终可编辑、可拒绝。停止、回到地面、删除、重命名、跳过都是一手数据——不是失败。

试点设计

小规模、被试内、带消融。

项目面向实验室的版本
研究类型被试内可行性试点;重复会话;非临床被试;不作治疗声称。
样本依据N = 5–12 只用于 Phase 0 可行性:可用性、数据质量估计、不良体验发现与方差估计。如果可行性证据支持扩大,后续将需要合作实验室通过 IRB 与招募基础设施支持更大样本。
主结果个体内探针前预测误差跨会话的变化。
次结果命名稳定性、感质细粒度、解释自主性事件、完成率、不适/停止事件。
信号先自我报告;其次 HRV/RMSSD、EDA/SCR、呼吸、体温、动作;EEG/fNIRS 可选、视实验室而定。
控制/消融最低配置:pause-only vs. 完整 BCI-HRP 协议。推荐配置:A) 仅暂停;B) 声音 + 报告;C) 声音 + 预测 + 报告;D) 声音 + 预测 + 报告 + 可编辑 AI 命名。可穿戴 / EEG / fNIRS 扩展保持可选。条件顺序应尽可能 counterbalanced,以降低顺序效应与疲劳效应。
分析混合效应模型、事件锁定的描述性分析、消融比较、缺失数据报告、伪迹报告、个体曲线。
混淆控制 — 刺激熟悉度 vs. 自我校准

由于声音文件固定且重复,预测误差的下降可能只是记住了某个片段,而非自我校准的提升。要区分二者:(1) 在同一族中用未听过的刺激测预测(迁移测试);(2) 仅分析首次暴露的预测误差;(3) 在模型中纳入"新/熟"因子。命名稳定性带有同样的熟悉度混淆,并用同样的控制来报告。

仅暂停对照

协议必须显示出超过“暂停本身”的价值。如果增益完全可由 pause-only 条件解释,则结果不能被解释为声音探针或可编辑命名带来的校准。为保证基线可比,每个暂停条件使用固定指令,例如:“请暂停你正在做的事并安静坐着;可以闭眼;不要使用手机,也不要刻意冥想或做任何特别的事——只是停止刚才的任务。”

会话流程

T0

知情同意

声音族、强度上限、停止方式、数据共享,以及是否允许 AI 摘要。

T1

基线

短暂休息加自我报告;可选生理基线。

T2

预测

使用者在声音之前预测状态变化的方向与强度。

T3

探针

固定、有记录、带事件标记的声音文件,停止控件始终可见。

T4

报告

使用者报告身体、情绪、知觉、行动倾向与意义。

T5

重命名

使用者接受、编辑、拒绝、重命名、删除或退出。

阶段路径

Phase 0

仅软件

Sound-QDR 协议:预测、报告、重命名与自主性日志——无硬件。

Phase 0b

扩大样本后续研究

如果 Phase 0 可行,用合作实验室支持的更大样本测试构念稳定性与效应估计。

Phase 1

加入可穿戴代理

用现有设备测 HRV、EDA、呼吸、动作与体温。

Phase 2

可选 EEG / fNIRS

仅当实验室条件与数据质量证明合理时,才作为"聆听态"扩展。

风险与证伪标准

第一项研究的目标是暴露失败模式,不是声称有效。

BCI-HRP 只有把风险写清楚才会更强。Phase 0 试点应先回答:这个协议是否可理解、是否安全、是否能产生有结构的数据——以及构念在哪里失败、缩小或需要重命名。

风险 01

任务学习,而非自我校准

预测误差下降可能只是参与者学会了任务或记住了声音片段。需使用新/熟刺激、迁移测试、首次暴露分析与研究后猜测检查。

风险 02

需求特征

参与者可能猜到协议希望他们如何回答。需使用中性提示、尽可能盲化声音族标签,并把"不知道/没有变化"视为有效数据。

风险 03

效应很小或高度异质

声音探针效应可能很小、不稳定或强烈因人而异。Phase 0 应估计方差和可用性,而不是证明疗效或有效性。

风险 04

生态效度有限

固定、可重复的声音提高了控制力,但可能不代表日常聆听。后续阶段可测试个性化声音、日常情境或实验室外迁移。

风险 05

代理信号可能没有额外价值

EEG、fNIRS、HRV、EDA 或动作信号可能无法比第一人称报告解释更多。如果如此,它们应继续作为可选代理,而非必需证据。

风险 06

构念可能需要重写

合作 PI 可以收窄、拆分、重命名或否决"自我校准"或"人类回归"。这应被视为构念校准,不是项目失败。

风险 07

普通暂停可能解释增益

如果 pause-only 产生与完整协议相同的变化,BCI-HRP 不应声称声音探针或可编辑命名具有额外价值。协议必须显示出普通暂停或书写之外的额外结构。

停止条件

如果参与者无法理解预测/报告任务,自主性控件很少被使用或被误解,不适明显升高,或预测变化在迁移测试中崩塌,则应先修订协议,再考虑加入传感器或扩大样本。

构念效度策略

预测误差改善只是候选指标,不是充分证据。

本协议不能把“预测更准”直接等同于“自我校准”。自我校准的解释需要预测、描述、命名、行动、契合性与自主性多维证据收敛。

主候选指标

预测误差变化

探针前预测状态向量 vs. 探针后报告状态向量。有用,但单独不足够。

三角验证

体验结构指标

感质细粒度、状态命名稳定性、行动倾向清晰度、对象关系报告、情境 / 价值契合性。

自主性效度

拒绝 / 重命名 / 编辑 / 删除

如果参与者只是无脑接受 AI 标签,协议可能测到的是顺从,而不是解释自主性。

怀疑检查

需求特征探针

研究后询问参与者认为实验在测什么,以及 AI 标签或重复声音是否影响了他们的回答。

解释边界

如果预测误差改善不能迁移到未听过的新刺激,不能伴随更细致的现象学报告,或可被怀疑检查解释,则结果应解释为任务学习、刺激熟悉度或需求特征——而不是自我校准。

决策规则

在数据采集前固定。自我校准解释需要同时满足三项:(a) 个体内预测误差变化;(b) 迁移到未听过的新刺激——必要条件;以及 (c) 至少两个三角验证指标收敛(感质细粒度、命名稳定性、行动倾向清晰度、对象/契合性、自主性修订)。如果迁移失败,或少于两个三角验证指标收敛,则结果报告为任务学习、熟悉度或需求特征,而非自我校准。对于 Phase 0,这些规则用于指导解释与下一步决策,不构成有功效力的有效性声称。

操作性定义

让"人类回归"可测量,而不把它还原成一个隐藏信号。

主

自我预测误差

探针前的预测向量 vs. 探针后的报告向量。候选维度:紧张、清晰、身体回归、动作冲动、情绪压力、安全。方向匹配 + 归一化绝对误差。

次

命名稳定性

在重复的声音族中,使用者是否复用、提炼或稳定其个人标签,而不被拉进 AI 的标签。

构念

感质细粒度

从笼统词("好/坏")转向有区分度的身体-情绪-知觉-行动描述。

自主性

解释自主性

停止、回到地面、删除、拒绝、重命名、跳过、编辑都记为一手事件,而非 UI 附属。

情绪-行动校准维度

情绪被测量为一个五维结构,而非隐藏的内在指纹。

维度捕捉什么候选字段
身体感受呼吸、紧张、沉重、温热、麻木、疲惫、放松或身体回归。body_tension, body_return, breath_note, movement_note
感受质地体验的纹理与基调:压力、明亮、平淡、尖锐、温热、漂移、清晰。felt_quality, qualia_terms, emotional_pressure, clarity
行动倾向该状态让人想做什么:靠近、回避、停下、书写、说话、休息、移动、联系、求助。action_tendency, action_readiness
对象/关系情绪"关于"什么:某人、任务、地点、身体部位、未来、丧失、威胁、机会或价值关系。material_object, formal_object, core_relational_theme
契合性该感受是否契合情境、对象与价值关系——而非是否"正面"。emotion_fit, situation_fit, value_object_fit, affordance_update
测量边界

合作实验室能获得什么

一个低风险试点、一套可复用协议、一个可发表问题。

合作实验室不是被要求验证一个成品发明,也不是一开始就承诺大规模研究。这个合作从一个低风险的 Phase 0 研究机会开始:仅软件、非临床、带明确失败条件、带仅暂停基线,并能为 HCI、个人信息学、情感计算或音乐认知提供一个以参与者为中心的新研究问题。如果 Phase 0 可行,合作可以分阶段推进到由实验室支持的更大样本后续研究。

相关工作定位

与情感 BCI、神经反馈相邻,但以"参与者自有的校准"为中心。

BCI-HRP 不声称声音、AI 或神经信号本身是新的。它的贡献在于协议的编排:声音作探针、参与者先预测、现象学报告、可编辑命名,以及把自主性事件当作数据。

相邻领域

情感 BCI

常用神经/生理/行为信号去估计、分类或适应情感状态。BCI-HRP 转而问:参与者如何预测、报告、修订并拥有"感受到的状态"的意义。

相邻领域

脑机音乐接口(BCMI)

常把脑或情感信号映射为音乐反馈/生成。BCI-HRP 把声音首先当作校准的受控探针,而非自动的情绪调控输出。

相邻领域

EEG / fNIRS 音乐-情绪研究

常通过效价、唤醒、注意或神经/血流特征分析对音乐的反应。BCI-HRP 增补第一人称预测、行动倾向、对象关系与契合性,作为校准材料。

相邻领域

神经反馈/生物反馈

常训练用户走向平静、专注或一致性。BCI-HRP 不为单一"好状态"优化;它检验参与者能否更好地理解并修订自己的情绪-行动结构。

差异化边界

不应把项目仅描述为"音乐 + BCI + 情绪调节"——那个空间已经很拥挤。BCI-HRP 更好的定位是:面向非侵入式情感 BCI 的、以参与者为中心的校准协议;可选的 EEG、fNIRS、HRV、EDA 始终是"使用者自有解释工作流"内部的代理,而非凌驾于"隐藏情绪"之上的权威。

安全交互流程

伦理必须表现为交互,而不只是原则。

1

同意门

使用者选择声音族、强度上限、停止方式、数据共享,以及是否允许 AI 摘要。

2

常驻控件

停止、回到地面、降低音量、跳过、删除——在播放前、中、后始终可见。

3

自动暂停触发

不安全的自我报告、反复的停止手势、无响应、不适快速上升,或实验室定义的生理阈值。

4

回到地面模式

淡出声音、抑制解释、显示落地提示、提供研究者问询。

5

不是危机系统

这不是紧急检测或治疗。高风险披露离开本协议,遵循实验室批准的人工安全流程。

UI 要求

始终可见的退出

停止键大于装饰性控件,键盘/触控可达,并被记为有效结果——而非失败。

UI 要求

触发后不作分析

若回到地面模式被触发,系统不产生深度解释——只总结使用者自己的安全动作。

UI 要求

自主性即数据

拒绝、重命名、停止、删除、跳过都是核心交互事件,不是附属。

问答

给合作者的简短回答。

本节把协议译成直接的问题,但不替代上面的正式研究简报。

这是治疗或诊断研究吗?

不是。BCI-HRP 是非诊断、非治疗的研究提案,不声称临床获益、危机检测或状态诱导。

BCI-HRP 只是冥想 App 吗?

不是。冥想通常训练注意、平静或调节。BCI-HRP 训练的是内感受预测、身体-情绪觉察与解释自主性:参与者预测、感受、命名、修订并拥有自己的状态描述。

它已经是可穿戴产品了吗?

不是。当前研究是 Phase 0 软件协议。长期转化路径可能包括 AI 耳机、腕带和可选头贴,但硬件是分阶段的,首项实验室研究不需要。

Phase 0 需要定制 BCI 硬件吗?

不需要。Phase 0 是软件协议:自我预测、声音探针、后报告、可编辑命名与自主性事件记录。定制 BCI 硬件属未来方向,首期试点不需要。

什么是自我校准?

自我校准被操作化为:探针前预测状态向量与探针后报告状态向量之间预测误差的个体内变化,辅以命名稳定性、感质细粒度与自主性事件等次级测量。

预测误差降低能证明自我校准吗?

不能。预测误差降低只是候选指标。自我校准解释需要感质细粒度、命名稳定性、行动倾向清晰度、对象/契合性报告、自主性事件、迁移到新刺激以及怀疑检查共同支持。

什么是"人类回归"?

"人类回归"指把参与者自己的解释保持在中心:AI 摘要可编辑、可拒绝、可删除;生理与神经信号始终是可选代理,而非隐藏真相。

BCI-HRP 是要让参与者感到正面情绪吗?

不是。本协议不以诱导正面情绪为目标。它检验参与者能否更好地识别一份感受如何与身体变化、行动倾向、对象关系与情境契合相关联。

什么是情绪-行动校准?

指追踪参与者能否跨会话更好地关联身体感受、感受质地、行动倾向、情绪对象与契合性。它是对预测误差分析的补充,不是临床结果声称。

什么是 Pause-QDR?

Pause-QDR 现在被定位为 Phase −1 内部仅暂停设计观察:用于提示需要设置仅暂停基线的非正式 alpha 反馈。它不作为正式研究数据或有效性证据呈现。

这和传统 BCI 有何不同?

传统 BCI 常强调外部任务控制或信号解码(如运动意图、视觉诱发控制)。BCI-HRP 聚焦对内源性体验的解释自主性与情绪-行动校准:人如何预测、命名、修订并拥有对自身内部状态描述的意义。

这不是已被情感 BCI 或神经反馈覆盖了吗?

部分相邻,但不等同。BCI-HRP 接近情感 BCI、脑机音乐接口、EEG/fNIRS 音乐-情绪研究与神经反馈。其差异化对象是参与者自有的校准:声音作探针、自我预测、现象学报告、可编辑 AI 命名,以及把拒绝/重命名/编辑/删除当作自主性数据。

合作 PI 能改动什么?

合作 PI 可主导实验设计、刺激选取、招募、安全审查、IRB 路径、分析方案与预注册,并可收窄、重命名、拆分或否决核心构念。

Phase −1 内部仅暂停设计观察

在正式研究之前,仍然需要控制仅暂停效应。

在寻求正式人类被试研究之前,我做过非正式的内部 alpha 观察,用来测试交互流程,并判断是否需要设置仅暂停条件。这些观察不作为研究数据呈现,不作为有效性证据,也不作为可发表数据集的一部分。

实际设计教训很简单:正式 Phase 0 Sound-QDR 研究应包含仅暂停基线,因为普通暂停本身可能改变身体觉察、情绪报告和任务投入。

IRB 边界

正式 Phase 0 研究将在招募或数据采集前获得适当的 IRB 判定或批准。任何此前的内部观察都只应被视为协议修订的设计反馈,而不是初步人类被试研究证据。

工作论文

学习分析中的解释性自决

探讨学习者能否保有对"关于自己的分析"的解释权,而不被系统标签所定义。直接驱动了本协议中可编辑命名与解释自主性的设计。

状态:撰写中 · 草案可应请求提供。

研究概念

声学熟悉度记忆回归(AFMR)

一个针对老年人"先熟悉、后回忆"的窄范围、非指涉性声学线索概念,带反暗示保障——是同一"声音探针 + 解释自主性"逻辑的聚焦应用。

状态:概念笔记发展中 · 草案可应请求提供。

合作模式

这是分工合作,不是请求实验室背书。

我带来什么

  • "人类回归 / 自我校准"构念与解释自主性交互逻辑。
  • 声音探针刺激设计与感质命名提示框架。
  • 协议软件实现、数据 schema 与可导出的会话日志。
  • 内部仅暂停设计记录与定性编码框架。
  • 数据分析、可视化与论文初稿的主导工作。

合作实验室提供什么

  • 在研究设计、统计、条件平衡与预注册上的方法学指导。
  • 安全审查,以及对适当 IRB 路径或判定的指导。
  • 如果正式研究在实验室机构语境下推进,提供 faculty oversight。
  • 判断构念是否应被收窄、重命名、拆分或否决。
  • 提供合规招募路径的指导,但 Phase 0 初始试点不要求实验室承担招募劳动。
  • Phase 0b 或后续阶段:只有在双方同意扩大时,才讨论招募基础设施、物理空间、设备或 RA 支持。

第一步请求

一次 15 分钟的契合通话。

我希望进行一次 15 分钟的契合通话,判断 BCI-HRP 是否应被收窄为 Phase 0 的 HCI、个人信息学、音乐认知、情感计算或可穿戴传感可行性研究。

这不是经费请求、硬件请求,也不是请求实验室一开始就提供场地、设备、RA 支持或承诺大样本。对于初始 Phase 0 可行性试点,我会在适当 IRB 判定或批准下承担实现、排期、会话运行、数据处理、分析与初稿撰写。

第一次通话的目标是测试契合度、澄清方法与合规范围,并决定该构念是否应被收窄、重命名或否决。如果 Phase 0 可行,后续 Phase 0b 才讨论实验室是否支持扩大样本。

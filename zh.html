BCI - The Human Return Protocol

一个面向实验室合作的研究协议：测试声音探针、自我预测、可编辑命名与可穿戴/神经代理，是否能提升个体内自我校准。

当前试点：Sound-QDR 协议优先；现有实验室设备其次；未来三件套硬件最后。它不是消费级 BCI 产品，也不是诊断、治疗或状态诱导系统。

当前试点：Sound-QDR未来愿景：AI 耳机 + 头贴 + 腕环不诊断 · 不治疗 · 不诱导状态

一个可证伪主张

声音探针 + 自我预测 + 可编辑命名协议，是否能提升个体内自我校准？自主神经/神经代理是否比仅自评提供额外解释价值？

研究定位：这是可行性与构念校准，不是临床验证。“感质对齐”和“Human Return”是候选构念，可以在预注册前由合作实验室缩小、改名、否定或修正。

为什么值得做

现有系统往往不是过度解释用户，就是过度压扁体验。

情绪工具常把人压进 calm / stressed 这样的粗标签；消费级神经科技容易对弱生理代理做过度解释；音乐工具又常只做 mood playlist，而不处理自我预测、解释权与安全退出。BCI-HRP Lab 做的是中间缺口：声音成为可控探针，用户先预测，生理信号只是代理，AI 解释始终可编辑、可拒绝、可删除。

研究缺口
Gap 01

过度依赖粗情绪标签，缺少感质颗粒度。

Gap 02

传感器权威过强，用户解释权不足。

Gap 03

声音常被当成 mood control，而不是可证伪探针。

Gap 04

伦理常停留在原则，缺少交互级安全设计。

科学版层级模型

保留结构复杂度，但把它翻译成科学上可读的语言。

Layer 01 — 声音事件

固定音频文件，具有已记录的声学属性与事件标记。

Layer 02 — 感官接触

声音如何通过听觉、身体共振、呼吸耦合与注意进入人。

Layer 03 — 自主神经 / 身体反应

HRV、EDA、运动、皮温与身体自评，作为个体内反应轨迹。

Layer 04 — 情绪构成

身体变化如何被体验成压强、释放、激活、安全或过载。

Layer 05 — 认知预测

用户在播放前以为会发生什么，以及预测误差是否下降。

Layer 06 — 意义生成

记忆、未来方向、自传性共振与价值意义笔记。

Layer 07 — 解释自主

接受、编辑、拒绝、重命名、删除、停止、落地、跳过。

科学锚点
Interoception

身体内部状态感知与解释。

Allostasis

调节与恢复，而不是静态平衡。

Constructed Emotion

情绪不是从信号中直接读出。

Predictive Processing

声音前预测误差是可操作目标。

Personal Informatics

收集、反思、行动，但不把数据当绝对真相。

协议快照

PI 应该能在三分钟内判断是否值得一次 fit call。

研究单位

Session，不是设备

一次声音反应 session：预测、声音探针、报告、可编辑命名与解释自主事件。

主要测试

预测误差

个体内声音前预测误差是否跨 session 变化，并能否迁移到未听过的新刺激。

安全原则

解释前先保留自主

AI 总结始终可编辑、可拒绝。停止、落地、删除、重命名、跳过是主要数据。

试点研究设计

小样本、个体内、可消融。

项目设计
研究类型个体内可行性试点；重复 session；非临床参与者；不做治疗声明。
样本依据N=5–12 只支持可用性、数据质量、负面交互发现，以及为后续功效研究估计方差。
主要结果跨 session 的个体内声音前预测误差变化。
次要结果状态命名稳定性、感质颗粒度、解释自主事件、完成率、不适/停止事件。
信号自评优先；HRV/RMSSD、EDA/SCR、呼吸、皮温、运动其次；EEG/fNIRS 可选且由实验室条件决定。
消融仅自评；固定声音 + 自评；腕 + 声音；头 + 声音；完整三件套；静默/中性场。
操作化定义

让“人类返回”可测，但不把它压扁成隐藏信号。

自我预测误差

声音前预测向量与播放后报告向量比较。候选维度：紧张度、清晰度、身体返回、行动冲动、情绪压强、安全感。

感质颗粒度

从“舒服/难受”的泛化词，转向更细的身体—情绪—感知—行动描述。

状态命名稳定性

重复声音条件下，用户是否能复用、细化或稳定个人标签，而不是被 AI 标签强行牵引。

解释自主

停止、落地、删除、拒绝、重命名、跳过、编辑是主要数据事件，不是 UI 附属品。

技术可行性

试点不应依赖自研硬件。

领域现实可行版本边界
声音探针冻结、可复现、有 loudness、事件标记、来源与授权记录的音频。不是普适声音处方。
可穿戴使用现有研究可接受设备记录 HRV、EDA、呼吸代理、皮温、运动。只做个体内代理，不推断隐藏真相。
EEG/fNIRS可选扩展，不是核心证明。只作为聆听状态代理，不做意义解码。
AI 角色可选生产辅助和可编辑总结器。AI 标签可拒绝，不被当成真相。

Pause-QDR 静坐暂停前测数据

在声音探针之前，必须先测量“暂停本身”。

这组约 10 人静坐暂停探索性数据，不是 BCI-HRP 或 Sound-QDR 有效性证据。它作为 Phase -1 可行性材料：说明参与者可以完成前后自评、产生丰富感质笔记，并暴露任务需求、环境、脱离手机和纯暂停效应等关键混淆因素。

数据类型

Pause-only baseline

暂停日常任务约 10–12 分钟，记录前后心情、专注、身体状态评分与开放式笔记。

描述性信号

身体放松最明显

描述性看，N=9 的平均身体放松改善约 2.11；心情和专注变化异质性更高，不应夸大。

设计含义

必须加入 pause-only baseline

未来 Sound-QDR 试点必须区分暂停效应、环境效应、任务需求效应、声音效应与解释自主效应。

安全交互流程

伦理必须落实为交互，而不能只停留在原则声明。

同意门槛

用户选择声音家族、强度上限、停止方式、数据分享和 AI 总结许可。

持续控制

Stop / Ground / Volume Down / Skip / Delete 始终可见。

自动暂停

自评不安全、重复停止、不回应、不适上升或实验室阈值触发暂停。

非危机系统

高风险表露退出协议，进入实验室批准的人类安全流程。

PI 外联策略

不要用同一封通用邮件发给所有实验室。

项目有同一个协议核心，但不同实验室应该看到不同入口。外联邮件应匹配该实验室的方法学归属。

邮件模板见：pi_email_variants.md

合作模型

不是请实验室执行一个已经封闭的系统。

我带来

  • Human Return / QDR 构念与用户界面逻辑。
  • 感质命名提示与解释自主设计。
  • 声音探针交互原型与叙事设计。
  • AI 可编辑、可拒绝、可删除的伦理结构。

实验室带来

  • 方法学严谨、IRB 路径和安全审查。
  • 传感器选择、信号处理和伪迹标准。
  • 招募、统计建模和预注册。
  • 缩小、改名、否定或拆分假设的权力。
第一请求

一次 15 分钟的契合对话。

第一次对话不是融资请求，也不是硬件请求，只是判断这个方向更适合成为软件协议、可穿戴可行性试点、HCI 安全研究，还是音乐认知/神经美学项目。

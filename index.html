Lab Collaboration Brief // 实验室合作入口

BCI - The Human Return Protocol

A lab-facing protocol for testing whether sound probes, self-prediction, editable naming, and wearable/neural proxies can improve within-person self-calibration.

一个面向实验室合作的研究协议：测试声音探针、自我预测、可编辑命名与可穿戴/神经代理，是否能提升个体内自我校准。

Current pilot: Sound-QDR protocol first. Existing lab equipment second. Future triadic hardware last.

One Testable Claim // 一个可证伪主张

Does a sound-probe + self-prediction + editable naming protocol improve within-person self-calibration, and do autonomic/neural proxies add explanatory value beyond self-report alone?

声音探针 + 自我预测 + 可编辑命名协议，是否能提升个体内自我校准？自主神经/神经代理是否比仅自评提供额外解释价值？

Research stance: this is feasibility and construct calibration, not clinical validation. “Qualia alignment” and “Human Return” are candidate constructs that can be narrowed, renamed, rejected, or revised by a partner lab before preregistration.

研究姿态：第一项研究是可行性与构念校准，不是临床验证。“感质对齐”和“Human Return”是候选构念，可以在预注册前由合作实验室缩小、改名、否定或修正。

Not a device pitch

Protocol first

Custom hardware is not required for the first study. Use fixed sound probes, self-report, interaction logs, and existing lab sensors.

第一项研究不依赖自研硬件。先用固定声音探针、自评、交互日志与实验室现有传感器。

Not hidden truth

Proxies, not mind-reading

HRV, EDA, motion, and optional EEG/fNIRS are physiological proxies; they do not reveal meaning, trauma, or “true emotion.”

HRV、EDA、运动、可选 EEG/fNIRS 都是生理代理；它们不揭示意义、创伤或“真实情绪”。

Not a finished belief system

Co-designable

The lab may change hypotheses, metrics, stimuli, hardware path, and construct names before any formal study.

实验室可以在正式研究前修改假设、指标、刺激材料、硬件路径与构念名称。

Session Flow // 单次流程
T0 Consent

Sound family, intensity ceiling, stop method, data sharing, AI summary permission.

T1 Baseline

Short rest + self-report; optional physiological baseline.

T2 Predict

User predicts direction and intensity of state change before the sound.

T3 Probe

Fixed, documented sound file with event markers and stop controls.

T4 Report

User reports body, affect, perception, action tendency, meaning.

T5 Rename

User accepts, edits, rejects, renames, deletes, or exits.

Operational Definitions // 操作化定义

Make “Human Return” measurable without reducing it to a hidden signal.

让“人类返回”可测，但不把它压扁成隐藏信号。

Primary

Self-prediction error

Pre-probe predicted vector compared with post-probe reported vector. Candidate dimensions: tension, clarity, body return, movement urge, emotional pressure, safety. Direction match + normalized absolute error.

声音前预测向量与播放后报告向量比较。候选维度：紧张度、清晰度、身体返回、行动冲动、情绪压强、安全感。计算方向匹配与归一化绝对误差。

Secondary

State-naming stability

Across repeated sound families, does the user reuse, refine, or stabilize personal labels without being forced into AI labels?

在重复声音条件下，用户是否能复用、细化或稳定个人标签，而不是被 AI 标签强行牵引？

Construct

Qualia granularity

Shift from generic terms (“good / bad”) to differentiated bodily-affective-perceptual-action descriptions.

从“舒服/难受”的泛化词，转向更细的身体—情绪—感知—行动描述。

Agency

Interpretive agency

Stop, Ground, Delete, Reject, Rename, Skip, and Edit are primary events, not UI afterthoughts.

停止、落地、删除、拒绝、重命名、跳过、编辑是主要数据事件，不是 UI 附属品。

What Can Fail // 失败判准

The project becomes stronger when failure is visible.

失败被写清楚，项目才更可信。

Usability failure

User cannot complete prediction, playback, report, rejection, or exit flow.

用户无法完成预测、播放、报告、拒绝或退出流程。

Prediction failure

Prediction error does not decrease, or users become more confused after exposure.

预测误差无下降，或暴露后更混乱。

Naming failure

Labels remain generic, unstable, or AI-led across repeated sessions.

标签持续泛化、不稳定，或被 AI 牵着走。

Safety failure

Users feel guided, diagnosed, pushed by sound, or unable to stop.

用户感到被诱导、被诊断、被声音推着走或无法退出。

Signal failure

Synchronization failure, high artifact load, or missing data prevents analysis.

同步失败、伪迹太高、缺失过多，无法支持分析。

Added-value failure

Wearable/neural proxies add no explanatory value beyond self-report.

可穿戴/神经代理没有比自评提供更多解释价值。

Future Hardware Boundary // 未来硬件边界

The triadic product vision remains: AI headphones for timed acoustic output, head patch for optional listening-state proxies, wristband for autonomic/body witness, and AI-QDR for interpretive return. But the first study should prove the protocol can work before the devices become the project.

三件套产品愿景保留：AI 耳机负责定时声音输出，头部贴片负责可选聆听状态代理，腕环负责自主神经/身体见证，AI-QDR 负责解释返回。但第一项研究必须先证明协议有价值，而不是让设备先吞掉项目。

Safety Interaction Flow // 安全交互流程

Ethics must appear as interaction, not only as principle.

伦理必须落实为交互，而不能只停留在原则声明。

1. Consent gate

User chooses sound family, intensity ceiling, stop method, data sharing, and whether AI summaries are allowed.

用户先选择声音家族、强度上限、停止方式、数据分享与是否允许 AI 总结。

2. Persistent controls

Stop, Ground, Volume Down, Skip, and Delete remain visible before, during, and after playback.

播放前、中、后始终显示停止、落地、降音量、跳过、删除。

3. Auto-pause triggers

Self-report unsafe, repeated stop gestures, non-response, rapid discomfort increase, or lab-defined physiology threshold.

自评不安全、重复停止、不回应、不适评分快速上升，或实验室定义阈值。

4. Grounding mode

Fade out sound, suppress interpretation, show grounding prompt, offer researcher check-in.

声音淡出，暂停解释，显示落地提示，提供研究人员 check-in。

5. No crisis system

This is not emergency detection or therapy. High-risk disclosures leave the protocol and follow lab-approved human safety procedures.

这不是危机检测或治疗系统。高风险表露退出协议，进入实验室批准的人类安全流程。

UI requirement

Always-visible exit

Stop must be larger than decorative controls, keyboard/touch accessible, and logged as a valid outcome rather than failure.

Stop 必须比装饰控件更显眼，支持键盘/触控，并作为有效结果记录，而不是失败。

UI requirement

No post-trigger analysis

If grounding mode is triggered, AI-QDR does not produce deep interpretation. It only summarizes user-owned safety actions.

进入落地模式后，AI-QDR 不做深度解释，只总结用户拥有的安全行动。

UI requirement

Agency as data

Reject, rename, stop, delete, and skip are primary interaction events, not UI afterthoughts.

拒绝、重命名、停止、删除、跳过是核心交互事件，不是界面附属品。

First Ask // 第一请求

A 15-minute fit call.

一次 15 分钟的契合对话。

The first meeting is not a funding ask and not a hardware ask. It is a fit call to decide whether this should become a Phase 0 software protocol, a wearable feasibility pilot, an HCI safety study, or a music-cognition / neuroaesthetics project.

第一次对话不是融资请求，也不是硬件请求。它只是判断这个方向更适合成为 Phase 0 软件协议、可穿戴可行性试点、HCI 安全研究，还是音乐认知/神经美学项目。
